# Engineering Handbook

Canonical engineering guide for AAES-OS contributors and maintainers.

## Summary

- Intent before generation
- Evidence before optimization
- Governance before execution
- Memory before mutation
- Measurement before scaling

## Runtime model

AAES-OS is organized around human intent, mission direction, agent council review, evolution, generation, evaluation, governance, memory, learning, and deployment.

## Core services

- Prompt Compiler
- Genome Service
- Image Generation Service
- Vision Evaluation Service
- Governance Service
- Memory Service
- Evolution Service
- Dashboard API

## Roadmap posture

v1.x is the stable baseline.
Later majors expand scope only when validated by implementation and governance review.
