# Canonical Replay & Evidence Contract (CREC)

CREC is the shared constitutional interface every repository in the stack should expose.

## Purpose

Give every repository the same evidence vocabulary while allowing each implementation to remain independent.

## Required Fields

- Intent
- Authority
- Evidence
- Verification
- Compliance
- Truth Boundary
- Replay Record
- Audit Trail
- Failure Path
- Proof & Challenge Surface
- Failure Contracts
- Observability
- Proof Surface Level (P0-P5)
- Constitutional Maturity
- Commercial Readiness

## Constitutional Rule

No repository may claim a constitutional state that is higher than the evidence on its proof surface.

## How to use it

Each repository should document the contract fields in its README, scorecard, or equivalent constitutional artifact, then back those fields with fresh evidence and explicit challenge evidence.

## Traceability

CREC claims should trace upward to the constitutional laws of intelligence and downward to repo-specific evidence artifacts.

## Failure Path

If evidence is missing, stale, or contradictory, the repository must say so explicitly and downgrade the claim instead of inflating maturity.

## Challenge Rule

Every repository claim should state both the evidence that supports it and the evidence that could invalidate it.
