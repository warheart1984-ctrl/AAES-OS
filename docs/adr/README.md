# Architecture Decision Registry

Every significant architectural change requires an ADR before implementation unless it is an emergency remediation. Emergency changes must receive a retrospective ADR before release promotion.

ADR states are `PROPOSED`, `ACCEPTED`, `REJECTED`, `SUPERSEDED`, and `DEPRECATED`. Accepted ADRs identify the constitutional principle, affected standards/specifications, runtime and conformance impact, evidence, trade-offs, and supersession relationships. The machine-readable registry is [`registry.json`](./registry.json).

An ADR records authorization to pursue a decision; it does not prove the implementation is correct or production-ready. Implementation, conformance, and release evidence remain separate traceability nodes.
