# Implementation Profile Worked-Example Policy

**Release family:** CRK-1 companion specification  
**Status:** Frozen  
**Depends on:** CIS Core v1.0, Implementation Profile Freeze Criteria  
**Stewardship:** CIS release maintainers  
**Constitutional sections:** 1-3  
**Informative sections:** 4

---

## 1. Policy

The implementation profile family SHALL have two frozen worked examples for now:

- `profile-government`
- `profile-research`

Additional `profile-*` artifacts, including `profile-finance`, SHALL remain `Draft` until they satisfy the Implementation Profile Freeze Criteria and have a completed evidence package.

## 2. Interpretation

This policy keeps the example set intentionally small so the freeze process stays clear and repeatable. The family remains capped at two frozen worked examples until a later profile earns frozen status through evidence.

The policy does not require any draft profile to freeze, and it does not prevent a future profile from freezing once the objective criteria are satisfied.

## 3. Governance rule

Any profile promotion to `Frozen` SHALL be supported by:

- the profile freeze criteria
- a traceability surface
- a complete evidence bundle
- a formal freeze notice
- a governed registry update

## 4. Informative note

This policy is a summary rule, not a replacement for the profile freeze criteria.
