# Implementation Profile: Research

**Release family:** CRK-1 companion specification  
**Status:** Frozen  
**Depends on:** CIS Core v1.0, Research OS, and Implementation Profile Freeze Criteria  
**Stewardship:** CIS release maintainers  
**Constitutional sections:** 1-6  
**Informative sections:** 7

---

## 1. Purpose

This implementation profile defines the research adaptation surface for CIS.

It is the worked example for the implementation profile freeze process and demonstrates how a profile may graduate from Draft to Frozen while remaining subordinate to CIS Core v1.0.

## 2. Scope

This profile applies to research-oriented deployments that require:

- reproducibility
- evidence-rich workflows
- traceable research outputs
- durable institutional memory
- clear linkage between research and constitutional governance

## 3. Adaptation model

The Research profile adapts CIS Core through the following governed surfaces:

- Research OS for Evidence -> Insights -> Ideas -> Actions
- Standards Traceability Matrix for requirement-to-evidence mapping
- Conformance Suite for constitutional, replay, receipt, trust, and acceptance validation
- Launch Readiness for evidence-based promotion gates
- Implementation Profile Freeze Criteria for the graduated freeze decision

The profile does not redefine constitutional obligations.

It specializes how those obligations are applied in research-heavy environments.

## 4. Constitutional boundaries

The Research profile SHALL:

- inherit CIS Core terminology
- preserve Research OS as the governing research workflow
- preserve the distinction between adaptation and constitutional authority
- remain traceable to the frozen CIS Core baseline
- remain traceable to the profile freeze criteria and release record

The Research profile SHALL NOT:

- redefine constitutional terms
- alter CIS Core requirements
- weaken replay, evidence, or stewardship expectations
- collapse profile guidance into constitutional governance

## 5. Freeze criteria satisfaction

The Research profile is frozen because it satisfies the objective profile freeze criteria:

1. The scope and boundary are explicit.
2. The profile inherits CIS Core terminology.
3. The profile maps to the research adaptation requirements it serves.
4. The profile has traceability links to the governing hierarchy and traceability surfaces.
5. The profile-specific evidence bundle is represented by the release artifacts listed in the freeze notice.
6. Build, test, and runtime-smoke expectations are governed by the profile freeze criteria.
7. Conformance is bounded to the frozen constitutional baseline.
8. Launch Readiness records the promotion as evidence-based.
9. The constitutional release record, merge readiness record, and evidence package are governed by the profile release surface.
10. No unresolved constitutional ambiguity, scope conflict, or traceability gap remains.
11. A formal freeze notice accompanies this profile.

## 6. Conformance

A conforming Research profile implementation SHALL:

- preserve the Evidence -> Insights -> Ideas -> Actions workflow
- maintain durable traceability between research objects
- keep research outputs linked to CIS Core and the traceability matrix
- support review without rewriting the constitutional baseline

Conformance SHALL be demonstrated through replayable evidence and stable object lineage.

## 7. Informative note

This profile is the worked example for the implementation profile freeze process.
It shows how a domain adaptation can graduate to Frozen without becoming a constitutional baseline.
