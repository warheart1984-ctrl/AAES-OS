# CIS Core Principles

**Release family:** CRK-1 companion specification  
**Status:** Frozen  
**Purpose:** Define the stable principles that CIS Core must preserve across companion specifications, implementation profiles, and future releases.

---

## 1. Principle of longevity and independent implementability

CIS Core must be specified so that an implementer who has never met the original authors can:

- read the specification without external interpretation
- build a conforming implementation
- verify conformance using published criteria
- evolve the implementation through the governance process
- do so years later without depending on founder knowledge

This principle keeps CIS legible across time, organizations, and vendor boundaries.

## 2. Principle of normative stability

Core terminology and constitutional requirements remain stable unless changed through the governed constitutional process.

## 3. Principle of inheritance over redefinition

Companion specifications, profiles, and documentation families inherit CIS terminology rather than redefining it.

## 4. Principle of model neutrality

The ecosystem must remain agnostic to any one reasoning engine, provider, or runtime implementation.

## 5. Principle of evidence before authority

Claims about conformance, readiness, and behavior must be backed by evidence, traceability, and tests.

## 6. Principle of governance continuity

Governance is part of the specification surface. It is not an external afterthought and not only a runtime concern.

## 7. Principle of independent evolution

Implementation profiles, runtime layers, and documentation layers may evolve independently as long as they preserve the constitutional core.
