# Implementation Profile: Government

**Release family:** CRK-1 companion specification  
**Status:** Frozen  
**Depends on:** CIS Core v1.0 and Implementation Profile Freeze Criteria  
**Stewardship:** CIS release maintainers  
**Constitutional sections:** 1-6  
**Informative sections:** 7

---

## 1. Purpose

This implementation profile defines the public-sector adaptation surface for CIS.

It is the second worked example for the implementation profile freeze process and demonstrates how a profile may graduate from Draft to Frozen while remaining subordinate to CIS Core v1.0.

## 2. Scope

This profile applies to government deployments that require:

- retention and records management
- procurement and approvals
- auditability and oversight
- access control and identity governance
- traceable public-service evidence

## 3. Adaptation model

The Government profile adapts CIS Core through the following governed surfaces:

- Implementation Profiles for domain adaptation boundaries
- Standards Traceability Matrix for requirement-to-evidence mapping
- Conformance Suite for constitutional, replay, receipt, trust, and acceptance validation
- Launch Readiness for evidence-based promotion gates
- Implementation Profile Freeze Criteria for the graduated freeze decision

The profile does not redefine constitutional obligations.

It specializes how those obligations are applied in public-sector environments.

## 4. Constitutional boundaries

The Government profile SHALL:

- inherit CIS Core terminology
- preserve governance continuity and auditability
- preserve explicit traceability to the frozen constitutional baseline
- remain traceable to the profile freeze criteria and release record
- preserve public-sector evidence obligations such as retention and oversight

The Government profile SHALL NOT:

- redefine constitutional terms
- alter CIS Core requirements
- weaken replay, evidence, or stewardship expectations
- collapse profile guidance into constitutional governance

## 5. Freeze criteria satisfaction

The Government profile is frozen because it satisfies the objective profile freeze criteria:

1. The profile scope and boundary are explicit.
2. The profile inherits CIS Core terminology.
3. The profile maps to the public-sector adaptation requirements it serves.
4. The profile has traceability links to the governing hierarchy and traceability surfaces.
5. The profile-specific evidence bundle is represented by the release artifacts listed in the freeze notice.
6. Build, test, and runtime-smoke expectations are governed by the profile freeze criteria.
7. Conformance is bounded to the frozen constitutional baseline.
8. Launch Readiness records the promotion as evidence-based.
9. The constitutional release record, merge readiness record, and evidence package are governed by the profile release surface.
10. No unresolved constitutional ambiguity, scope conflict, or traceability gap remains.
11. A formal freeze notice accompanies this profile.

## 6. Conformance

A conforming Government profile implementation SHALL:

- preserve retention, procurement, audit, and access-control constraints
- maintain durable traceability between government profile outputs and CIS Core requirements
- keep public-sector outputs linked to the traceability matrix
- support review without rewriting the constitutional baseline

Conformance SHALL be demonstrated through replayable evidence and stable object lineage.

## 7. Informative note

This profile is the contrasting worked example for the implementation profile freeze process.
It shows how a public-sector adaptation can graduate to Frozen without becoming a constitutional baseline.
