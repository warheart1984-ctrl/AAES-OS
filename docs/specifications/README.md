## What this folder is for

- Stable reference material for implementers and reviewers
- A single place to track the platform vision, architecture, governance, and release sequencing
- A source of truth for docs-site pages and follow-on implementation plans
- The first implementation slice that turns the extracted spec set into a concrete, documentation-first chunk
- The next sprint-sized backlog item after repository filing that tightens the public API reference language
- The next backlog item after API stabilization that formalizes claim-to-evidence traceability
- The next backlog item after governance traceability that cleans up the example application path
- The next companion-spec slice that formalizes the trust model and the governed control plane
- The AIOS node runtime specification that defines the smallest sovereign constitutional unit
- The unified knowledge substrate, cross-domain query language, package language, and portable replay artifact specification
- The managed-service contract that defines every constitutional operator adapter and its proof gates
- The SOCK machine-readable schema that mirrors the five frozen constitutional languages
- Live Coda and constitutional package surfaces: [@aaes-os/coda-doc](../../packages/coda-doc/src/index.ts), [@aaes-os/coda-runtime](../../packages/coda-runtime/src/index.ts), [@aaes-os/nova-coda](../../packages/nova-coda/src/index.ts), [@aaes-os/isl-runtime](../../packages/isl-runtime/src/index.ts), and [@aaes-os/gcre-sysmin](../../packages/gcre-sysmin/src/index.ts)
- Doc-forward corpus families that remain in the spec layer: CML-2, CVM-1, and The Voss Binding
